The repository is initially empty. If your component is going to use the Marvin framework, I suggest to use the Marvin Initializer in order to get an archetype of the new component pre-configured for Marvin A project has been created for the component where source code should be managed. Instructions to upload the first version of the component source code is explained on the front page of the empty repository. The members of the project for the component are by default inherited from the workspace of the application code. Members contributing to the specific component can be added by the application designer or any of their back-up.
